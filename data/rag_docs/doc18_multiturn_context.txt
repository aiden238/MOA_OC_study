멀티턴 대화 컨텍스트 관리

멀티턴 대화(Multi-turn Conversation)는 사용자와 에이전트가 여러 차례 주고받는 대화 형식으로, 이전 대화의 내용을 유지하면서 일관된 답변을 생성해야 합니다.

대화 상태 관리의 핵심 과제:
1. 의도 연속성: 여러 턴에 걸친 사용자 의도 추적
2. 대명사 해석: "그것", "그 방법" 같은 지시어 이해
3. 누적 지식: 이전 턴에서 언급된 정보 활용
4. 주제 전환 감지: 새로운 주제로 전환 시 상태 업데이트

대화 이력 관리 전략:

1. 전체 이력 유지(Full History)
- 모든 대화를 컨텍스트에 포함
- 장점: 완전한 맥락 유지
- 단점: 컨텍스트 윈도우 빠르게 소진

2. 롤링 윈도우(Rolling Window)
- 최근 N개 메시지만 유지
- 장점: 메모리 효율, 단점: 초기 정보 손실
- N = 5~10이 실용적 균형

3. 요약 + 최근 이력(Summary + Recency)
- 오래된 대화는 요약, 최근 N개는 원본 유지
- 가장 많이 사용되는 하이브리드 전략

4. 세션 슬롯(Session Slots)
- 중요한 정보를 구조화된 슬롯에 저장
- 예: {"user_name": "Kim", "project": "MOA", "current_task": "RAG 구현"}
- 슬롯 기반 대화 관리(Slot Filling Dialog)

세션 스토어 설계:
```python
class Session:
    session_id: str
    messages: list[Message]  # 전체 대화 이력
    summary: str             # 요약된 이전 내용
    metadata: dict           # 사용자 정보, 선호도 등
    last_active: datetime
```

컨텍스트 리셋 전략:
- 주제 전환 감지 시 이전 요약만 유지
- 사용자의 명시적 리셋 요청 처리
- 세션 만료 후 장기 메모리에 저장

MOA 웹 UI의 세션 관리:
- 세션 ID로 대화 이력을 서버 사이드에서 관리
- 요청마다 관련 이력을 컨텍스트에 포함
- 세션별 모델 설정과 실행 경로 저장
