메모리 계층 설계 (단기/장기/외부 메모리)

LLM 기반 에이전트 시스템에서 메모리는 크게 세 가지 계층으로 구분됩니다. 각 계층은 서로 다른 지속성, 접근 방식, 활용 목적을 가집니다.

1. 단기 메모리 (In-Context Memory)
- 위치: 현재 컨텍스트 윈도우 내
- 지속성: 세션 내에서만 유효
- 용량: 컨텍스트 윈도우 크기에 제한 (수천~수백만 토큰)
- 활용: 현재 대화 이력, 진행 중인 작업 상태
- 특징: 가장 빠른 접근, 세션 종료 시 소멸

2. 장기 메모리 (External Storage Memory)
- 위치: 데이터베이스, 벡터 스토어, 파일 시스템
- 지속성: 영구적 (명시적으로 삭제하지 않는 한 유지)
- 용량: 제한 없음
- 활용: 사용자 프로필, 학습된 선호도, 프로젝트 이력
- 특징: RAG를 통해 필요시 검색, 지연 시간 존재

3. 외부 메모리 (Tool/MCP Memory)
- 위치: 파일 시스템, API, 데이터베이스
- 지속성: 외부 시스템에 의존
- 용량: 무제한
- 활용: 코드 실행 결과, 웹 검색 결과, 파일 내용
- 특징: 도구 호출로 접근, 실시간 데이터 가능

메모리 계층 간 상호작용:
```
외부 메모리 (파일, DB) ←→ RAG/MCP ←→ 장기 메모리 (벡터 스토어)
                                              ↓ 검색
                              단기 메모리 (컨텍스트 윈도우)
                                              ↓
                                        LLM 추론
```

메모리 관리 패턴:
1. 기억하기(Remember): 중요 정보를 장기 저장소에 저장
2. 검색하기(Retrieve): 관련 정보를 컨텍스트에 포함
3. 잊기(Forget): 오래되거나 중요도 낮은 정보 제거
4. 압축하기(Compress): 긴 메모리를 요약해 공간 절약
5. 갱신하기(Update): 기존 메모리를 새 정보로 업데이트

에이전트 메모리 설계 고려사항:
- 어떤 정보를 언제 저장할 것인가?
- 검색 정확도를 어떻게 보장할 것인가?
- 민감한 정보는 어떻게 처리할 것인가?
- 메모리 용량 한계에 도달하면 무엇을 삭제할 것인가?
