# 05. 평가 지표 명세

## 개요

MOA 오케스트레이션 실험에서 사용하는 **품질 지표**와 **시스템 지표**를 정의한다.
모든 경로(Single / MOA / Full)의 결과는 동일한 루브릭과 지표로 평가된다.

---

## 품질 지표 (LLM Judge 루브릭)

LLM을 Judge로 사용하여 생성된 출력의 품질을 1~5점 척도로 채점한다.

| 항목 | 설명 | 1점 | 3점 | 5점 |
|------|------|-----|-----|-----|
| `clarity` | 읽기 쉬운가 | 혼란스럽고 이해하기 어려움 | 대체로 이해 가능 | 매우 명확하고 읽기 쉬움 |
| `structure` | 논리 구조가 명확한가 | 두서없고 구조 없음 | 기본 구조 존재 | 체계적이고 논리적 흐름 |
| `constraint_following` | 제약 조건을 지켰는가 | 제약 조건 무시 | 일부 준수 | 모든 제약 조건 완벽 준수 |
| `usefulness` | 실제로 도움이 되는가 | 무의미하거나 부정확 | 보통 수준으로 유용 | 매우 유용하고 실용적 |

**종합 점수:** 4개 항목의 산술 평균 (1.0 ~ 5.0)

---

## 시스템 지표

| 지표 | 단위 | 계산 방법 |
|------|------|----------|
| `total_latency_ms` | ms | 해당 run의 모든 trace latency_ms 합계 |
| `total_tokens` | tokens | prompt_tokens + completion_tokens 합계 |
| `total_cost_estimate` | USD | 모든 trace의 cost_estimate 합계 |
| `agent_count` | 개 | 호출된 에이전트 수 |

---

## 비교 축

| 비교 | 의미 |
|------|------|
| Single vs MOA | 오케스트레이션이 품질을 높이는가? |
| 비용 대비 품질 | 추가 비용 대비 품질 향상 비율이 정당한가? |
| 유형별 차이 | 어떤 태스크 유형에서 MOA가 효과적인가? |

---

## 평가 프로토콜 (교차 검증)

1. LLM Judge가 모든 벤치마크 케이스를 자동 채점
2. 사람이 12건 중 5건을 수동 채점
3. LLM 점수와 수동 점수의 상관관계(Pearson) 계산
4. **상관계수 ≥ 0.7** → LLM Judge 신뢰, 이후 자동 채점만 사용
5. **상관계수 < 0.7** → 루브릭 재조정 후 재실행

## 벤치마크 유형별 MOA 기대 효과

| 유형 | 코드 | 난이도 | MOA 기대 효과 | 측정 목적 |
|------|------|--------|--------------|----------|
| 단순 요약 | `summarize` | low | 최소 | MOA가 불필요한 케이스 확인 |
| 구조화 설명 | `explain` | medium | 중간 | Planner 태스크 분해 효과 측정 |
| 창의적 아이디어 | `ideate` | medium | 높음 | Draft 다양성 효과 측정 |
| 비판-재작성 | `critique_rewrite` | high | 높음 | Critic → Rewrite 루프 효과 측정 |
