컨텍스트 엔지니어링 개요

컨텍스트 엔지니어링(Context Engineering)은 LLM의 컨텍스트 윈도우(입력 공간)를 효율적으로 구성하고 관리하는 고급 기술 분야입니다. 단순히 프롬프트를 작성하는 것을 넘어, 모델이 최적의 판단을 내릴 수 있도록 컨텍스트 전체를 설계하는 관점입니다.

컨텍스트의 구성 요소:
1. 시스템 프롬프트: 에이전트의 역할과 지침
2. 대화 이력: 이전 메시지들
3. 검색된 문서(RAG): 외부 지식
4. 도구 결과(MCP/Function): 실행 결과
5. 현재 작업 컨텍스트: 진행 중인 태스크 상태
6. 제약 조건: 출력 형식, 토큰 한도 등

컨텍스트 엔지니어링의 핵심 과제:
1. 컨텍스트 윈도우 한계: 무한히 넣을 수 없음
2. 정보 우선순위: 어떤 정보를 앞에 넣을 것인가
3. 신선도 관리: 오래된 정보와 최신 정보의 균형
4. 중복 제거: 동일한 정보를 반복하지 않기
5. 관련성 필터링: 현재 태스크와 무관한 정보 제거

Lost in the Middle 문제:
연구에 따르면 LLM은 컨텍스트의 시작과 끝 부분 정보는 잘 활용하지만, 중간 부분 정보는 상대적으로 무시하는 경향이 있습니다. 따라서 중요한 정보는 컨텍스트의 앞이나 뒤에 배치해야 합니다.

컨텍스트 배치 전략:
- 중요도 순: 가장 중요한 지침 → 관련 문서 → 대화 이력 → 현재 질문
- 시간 순: 최신 정보를 뒤에 배치 (Recency 효과 활용)
- 계층 순: 전역 지침 → 도메인 지식 → 태스크별 지시

컨텍스트 엔지니어링 vs 프롬프트 엔지니어링:
- 프롬프트 엔지니어링: 개별 프롬프트의 표현 최적화
- 컨텍스트 엔지니어링: 전체 입력 공간의 구성 설계
- 컨텍스트 엔지니어링이 상위 개념으로, 더 시스템적 접근
