멀티 에이전트 오케스트레이션 패턴

멀티 에이전트(Multi-Agent) 시스템은 여러 LLM 에이전트가 협력하거나 경쟁하여 단일 에이전트보다 복잡한 태스크를 해결하는 아키텍처입니다. 다양한 오케스트레이션 패턴이 존재합니다.

핵심 패턴:

1. MOA (Mixture of Agents)
여러 초안 에이전트가 독립적으로 답변을 생성하고, 상위 에이전트가 이를 통합:
- 다양성 확보: 각 에이전트가 다른 관점으로 접근
- 병렬 처리: asyncio로 동시 실행
- 통합: Synthesizer가 최선의 요소 결합
대표 사례: 본 MOA_OC_study 프로젝트

2. Critic-Revision 패턴
에이전트가 출력하면 Critic이 평가하고 개선 피드백을 제공:
- Draft → Critic → Rewrite → Critic → ...
- Judge가 최종 허용/거절 결정
- 자기 개선(Self-improvement) 메커니즘

3. Debate 패턴
여러 에이전트가 서로의 주장을 반박하며 더 강한 논거 도출:
- 에이전트 A: 주장 1 제시
- 에이전트 B: 주장 1을 반박하고 주장 2 제시
- 에이전트 A: 반론 및 주장 강화
- Judge: 최종 판정

4. Blackboard 패턴
공유 메모리(Blackboard)에 여러 에이전트가 기여:
- 각 에이전트가 전문 분야 정보를 공유 공간에 기록
- 다른 에이전트가 이를 읽고 활용
- 협업적 문제 해결

5. Supervisor-Worker 패턴
슈퍼바이저가 태스크를 분해하고 워커 에이전트에 할당:
- 슈퍼바이저: 계획, 할당, 통합
- 워커: 특정 서브태스크 실행
- 슈퍼바이저가 전체 진행 상황 모니터링

패턴 선택 가이드:
- 단순 Q&A → Single path
- 다각도 분석 → MOA 패턴
- 반복 개선 → Critic-Revision
- 논쟁적 주제 → Debate
- 복잡한 장기 작업 → Plan-and-Execute + Supervisor-Worker

에이전트 수와 성능의 트레이드오프:
에이전트가 많을수록 비용과 지연이 증가하지만, 일정 수준까지는 품질도 향상됩니다. 일반적으로 3~5개의 다양한 에이전트가 최적의 비용-성능 균형을 제공합니다.
