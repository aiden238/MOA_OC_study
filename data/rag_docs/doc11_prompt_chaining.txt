프롬프트 체이닝 (Prompt Chaining)

프롬프트 체이닝은 복잡한 태스크를 여러 개의 단순한 프롬프트로 분해하고, 각 단계의 출력을 다음 단계의 입력으로 연결하는 기법입니다. 단일 프롬프트로는 처리하기 어려운 복잡한 작업을 단계적으로 해결할 수 있습니다.

기본 구조:
입력 → [프롬프트 1] → 중간 결과 1 → [프롬프트 2] → 중간 결과 2 → [프롬프트 3] → 최종 출력

MOA 파이프라인은 프롬프트 체이닝의 전형적 예시:
1. Router: "이 질문에 어떤 처리 경로가 필요한가?" → routing_decision
2. Draft Agents: routing_decision + 원본 질문 → 3개의 초안
3. Critic: 3개의 초안 → 비교 분석 리포트
4. Synthesizer: 초안 + 분석 → 최종 답변
5. Judge: 최종 답변 → 품질 평가 결정

체이닝 패턴 유형:
1. 순차 체이닝(Sequential): A → B → C → D
2. 병렬 체이닝(Parallel): A → [B, C, D] → E (asyncio.gather)
3. 조건부 체이닝(Conditional): A → if pass: stop, if rewrite: B → C
4. 반복 체이닝(Loop): A → Judge → if fail: Rewrite → Judge → ...
5. 계층 체이닝(Hierarchical): 플래너가 하위 에이전트에 분배

효과적인 체이닝 설계:
- 각 단계의 입출력 스키마를 명확히 정의
- 중간 결과를 검증하는 게이트 포함
- 실패 시 재시도 또는 폴백 경로 설계
- 각 단계가 독립적으로 테스트 가능하도록 설계

체이닝의 장단점:
장점: 복잡한 태스크 처리, 각 단계 최적화 가능, 디버깅 용이
단점: 지연 누적, 오류 전파, 비용 증가, 컨텍스트 손실 가능
