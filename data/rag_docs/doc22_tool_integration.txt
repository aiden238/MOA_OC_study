도구 통합 및 함수 호출 설계

LLM 에이전트는 텍스트 생성만으로는 한계가 있어 외부 도구와 통합하여 실제 작업을 수행합니다. 함수 호출(Function Calling), 도구 사용(Tool Use), MCP(Model Context Protocol) 등이 이를 구현하는 주요 방식입니다.

함수 호출 기본 구조:
1. 도구 정의: 이름, 설명, 파라미터 스키마를 JSON으로 정의
2. 모델 판단: LLM이 어떤 도구를 어떤 파라미터로 호출할지 결정
3. 도구 실행: 애플리케이션이 실제 함수를 실행
4. 결과 반환: 실행 결과를 LLM 컨텍스트에 포함
5. 최종 응답: 도구 결과를 바탕으로 최종 답변 생성

도구 정의 예시:
```json
{
  "name": "search_files",
  "description": "로컬 파일 시스템에서 파일을 검색합니다",
  "parameters": {
    "type": "object",
    "properties": {
      "path": {"type": "string", "description": "검색할 경로"},
      "pattern": {"type": "string", "description": "파일명 패턴"}
    },
    "required": ["path"]
  }
}
```

MCP(Model Context Protocol) 아키텍처:
MCP는 Anthropic이 제안한 표준화된 도구 통합 프로토콜:
- MCP 서버: 특정 기능을 제공하는 도구 서버
- MCP 클라이언트: LLM 에이전트 측에서 MCP 서버를 호출
- 표준 도구 타입: 파일 시스템, 웹 검색, 코드 실행, DB 조회 등

도구 설계 원칙:
1. 단일 책임: 하나의 도구는 하나의 잘 정의된 기능
2. 원자성: 실패 시 부분 실행 없이 완전 롤백
3. 멱등성: 동일 입력에 동일 결과 (재시도 안전)
4. 명확한 오류: 실패 원인을 이해할 수 있는 에러 메시지
5. 적절한 추상화: 너무 낮은 수준의 도구는 판단 오류 유발

도구 선택 전략:
- 라우터가 필요한 도구를 미리 결정
- 모델이 동적으로 도구를 선택 (ReAct 패턴)
- 특정 에이전트에 고정 도구 할당

비용과 리스크 관리:
- 부작용 있는 도구(파일 쓰기, 이메일 전송)는 사용자 확인 필요
- 읽기 전용 도구와 쓰기 도구를 명확히 구분
- 도구 호출 횟수 제한으로 무한 루프 방지
