시스템 프롬프트 설계 원칙

시스템 프롬프트(System Prompt)는 LLM의 전체 동작을 제어하는 고수준 지침으로, 사용자 메시지보다 먼저 처리되어 모델의 기본 행동 방식을 결정합니다. 에이전트 시스템에서 가장 중요한 설계 요소 중 하나입니다.

시스템 프롬프트의 핵심 구성 요소:
1. 역할 정의: 에이전트가 누구이며 무엇을 하는지
2. 목표와 우선순위: 달성해야 할 것과 중요도 순서
3. 행동 제약: 하지 말아야 할 것 (네거티브 지시)
4. 출력 형식: 응답의 구조와 포맷
5. 예외 처리: 불확실한 상황에서의 행동 지침

설계 원칙:
1. 최상위 지시를 먼저: 가장 중요한 제약을 앞에 배치
2. 긍정문 우선: "하지 마세요" 보다 "하세요" 형식
3. 구체적 예시 포함: 추상적 지시보다 예시가 효과적
4. 충돌 방지: 서로 모순되는 지시 없애기
5. 간결성: 길수록 초반 지시가 무시될 가능성 증가

시스템 프롬프트 레이아웃 예시 (MOA Draft Agent):
```
[역할]
당신은 분석적 초안 작성 에이전트입니다.

[목표]
주어진 질문에 대해 논리적이고 근거 기반의 초안을 작성합니다.

[행동 방식]
- 데이터와 사실에 기반한 분석 제공
- 주장마다 근거나 예시를 포함
- 구조적으로 정리된 문단 형식 사용

[제약]
- 추측이나 확인되지 않은 사실 포함 금지
- 500단어 이내로 작성
- 결론을 마지막 단락에 명확히 제시
```

시스템 프롬프트 vs 유저 메시지:
- 시스템 프롬프트: 영구적 행동 지침 (변경 빈도 낮음)
- 유저 메시지: 개별 요청 (매 호출마다 변경)
- 시스템 프롬프트의 제약이 유저 지시보다 우선

프롬프트 파일 관리 전략:
- 에이전트별로 별도 파일 관리 (예: draft_analytical.md)
- 버전 관리 시스템(git)으로 변경 이력 추적
- 테스트셋으로 변경 효과 측정 후 적용
