하네스 심화 기법 (레이어 패턴 및 컨텍스트 관리)

하네스 엔지니어링의 고급 패턴들로 복잡한 에이전트 시스템을 안정적으로 운영하는 방법입니다.

컨텍스트 계층 관리 (Context Layer Management):

레이어 1 - 불변 기반 컨텍스트 (Immutable Base):
- 에이전트의 핵심 역할과 절대 규칙
- 세션 전체에서 변경되지 않음
- 시스템 프롬프트 파일로 관리 (draft_analytical.md 등)
- 프롬프트 캐싱의 대상

레이어 2 - 세션 컨텍스트 (Session Context):
- 현재 세션의 사용자 의도와 진행 상태
- 세션 내에서 점진적으로 업데이트
- 대화 요약, 결정 사항, 선호도 포함

레이어 3 - 태스크 컨텍스트 (Task Context):
- 현재 실행 중인 특정 태스크의 상태
- 태스크 완료 시 폐기 또는 세션 컨텍스트로 승격
- RAG 검색 결과, 도구 실행 결과 포함

레이어 4 - 즉시 컨텍스트 (Immediate Context):
- 현재 LLM 호출에만 필요한 임시 정보
- 단일 호출 후 폐기
- 기타 에이전트의 출력, 중간 계산 결과

맥락 프로파일링(Context Profiling):
각 에이전트 호출에서 컨텍스트 품질을 측정:
```python
context_profile = {
    "total_tokens": 3500,
    "relevant_tokens": 2800,   # 현재 태스크 관련
    "cached_tokens": 1200,     # 캐시 히트
    "freshness_score": 0.85,   # 정보 최신성
    "redundancy_rate": 0.08,   # 중복 비율
}
```

에이전트 간 컨텍스트 전달 패턴:

1. 파이프라인 패턴 (각 단계의 출력을 다음 단계에 전달):
```python
drafts = await run_drafts(task)
critique = await critic.critique(drafts, original_prompt=task.prompt)
final = await synthesizer.synthesize(drafts, critique, original_prompt=task.prompt)
```

2. 방송 패턴 (모든 에이전트에 동일 컨텍스트):
```python
shared = {"task": task, "rag_context": rag_context}
results = await asyncio.gather(*[agent.run(shared) for agent in agents])
```

3. 선택적 전달 패턴 (에이전트별 필요한 컨텍스트만):
```python
analyst_ctx = {"task": task, "data": structured_data}
creative_ctx = {"task": task, "inspiration": examples}
```

하네스 상태 머신:
에이전트 시스템의 상태를 명시적으로 관리:
```
IDLE → ROUTING → DRAFTING → CRITIQUING → SYNTHESIZING
                                                ↓
                               JUDGING → PASSING or REWRITING
                                              ↓
                                        REWRITING → JUDGING
                                              ↓
                                        FINAL_OUTPUT
```

하네스 메트릭 수집:
각 레이어의 성능을 지속적으로 모니터링:
- 레이어별 평균 지연시간
- 컨텍스트 효율성 (관련 토큰 비율)
- 캐시 히트율
- 재시도 빈도
- 에스컬레이션 발생률
