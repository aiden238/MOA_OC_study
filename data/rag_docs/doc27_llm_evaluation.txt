LLM 시스템 평가 및 품질 측정

LLM 기반 시스템의 품질을 체계적으로 측정하고 개선하기 위한 평가 방법론입니다.

평가 지표 분류:

1. 품질 지표 (Quality Metrics)
- 정확성(Accuracy): 사실 오류 없는 답변 비율
- 관련성(Relevance): 질문에 실제로 답하는 정도
- 완전성(Completeness): 필요한 모든 정보 포함 여부
- 일관성(Consistency): 유사 질문에 일관된 답변
- 유창성(Fluency): 자연스럽고 읽기 쉬운 정도

2. 시스템 지표 (System Metrics)
- 지연시간(Latency): 응답 생성까지 소요 시간
- 처리량(Throughput): 단위 시간당 처리 요청 수
- 비용(Cost): 토큰당 비용, 호출당 비용
- 가용성(Availability): 시스템 정상 작동 시간 비율
- 오류율(Error Rate): 실패한 요청의 비율

평가 방법론:

1. 루브릭 기반 평가 (Rubric-based)
사전 정의된 기준으로 점수 부여:
```
점수 1-2: 질문과 무관하거나 심각한 오류
점수 3-4: 부분적으로 관련, 사실 오류 포함
점수 5-6: 관련성 있으나 불완전
점수 7-8: 좋은 답변, 사소한 개선 여지
점수 9-10: 완전하고 정확한 뛰어난 답변
```

2. LLM-as-Judge
다른 LLM이 평가자 역할을 수행:
- G-Eval: ChatGPT를 사용한 자동 평가
- 쌍방 비교(Pairwise Comparison): 두 답변 중 더 나은 것 선택
- 절대 평가(Absolute Scoring): 기준에 따른 점수 부여

3. 벤치마크 기반 평가
표준 데이터셋으로 객관적 비교:
- MMLU: 다영역 다지선다 문제
- TruthfulQA: 사실성 평가
- HumanEval: 코드 생성 능력
- 커스텀 벤치마크: 도메인 특화 테스트셋

4. 인간 평가 (Human Evaluation)
실제 사용자가 직접 평가:
- 가장 신뢰할 수 있으나 비용이 높음
- 표준화된 가이드라인 필요
- A/B 테스트 형식으로 비교

MOA vs Single 비교 평가:
- 동일 벤치마크 질문셋으로 두 경로 실행
- 품질 점수, 지연시간, 비용을 나란히 비교
- 개선율(%) = (MOA 점수 - Single 점수) / Single 점수 × 100

연속 모니터링:
- 프로덕션 트래픽에서 샘플링하여 지속 평가
- 품질 드리프트(Quality Drift) 조기 감지
- A/B 테스트로 프롬프트 변경 효과 측정
