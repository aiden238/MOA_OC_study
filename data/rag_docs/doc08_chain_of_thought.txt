Chain-of-Thought(CoT) 프롬프팅

Chain-of-Thought(CoT) 프롬프팅은 LLM이 최종 답변을 내기 전에 중간 추론 단계를 명시적으로 생성하도록 유도하는 기법입니다. 복잡한 수학 문제, 논리 추론, 다단계 의사결정에서 특히 효과적입니다.

기본 CoT 방식:
"단계별로 생각해보세요(Let's think step by step)."라는 문구를 추가하는 것만으로도 추론 품질이 크게 향상됩니다.

CoT 유형:
1. Zero-shot CoT: "단계별로 생각하세요"처럼 추론 유도 문구만 추가
2. Few-shot CoT: 추론 과정이 포함된 예시를 함께 제공
3. Auto-CoT: 자동으로 추론 예시를 생성해 사용
4. Tree of Thoughts (ToT): 여러 추론 경로를 탐색하고 최선을 선택

CoT가 효과적인 이유:
- 모델이 중간 계산 상태를 명시적으로 표현
- 복잡한 문제를 작은 하위 문제로 분해
- 오류가 생기면 추론 과정에서 발견 가능
- 긴 의존성(Long-range dependency)을 처리하는 데 유리

적용 예시:
문제: "사과 5개에서 3개를 먹고, 2개를 더 샀다. 남은 사과는?"
일반 프롬프트: 답만 출력 → 오류 발생 가능
CoT 프롬프트:
  "처음 5개 → 3개 먹음 → 5-3=2개 남음 → 2개 추가 구매 → 2+2=4개"
  → 정확한 답 도출

한계와 주의:
- 토큰 소비가 증가
- 추론 과정 자체에 오류가 포함될 수 있음
- 단순한 태스크에는 오히려 과적합(over-thinking) 발생 가능
- 추론 모델(o1, gpt-5 등)은 내부적으로 CoT를 수행하므로 별도 유도 불필요
