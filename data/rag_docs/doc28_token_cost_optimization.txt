토큰 비용 최적화 전략

LLM API 비용은 주로 입력 토큰과 출력 토큰 수에 비례합니다. 품질을 유지하면서 비용을 절감하는 전략이 중요합니다.

토큰 비용 구조 (2025년 기준 예시):
- GPT-4o-mini: 입력 $0.15/M, 출력 $0.60/M
- GPT-5-nano: 입력 $0.05/M, 출력 $0.40/M
- Gemini 2.5 Flash: 입력 $0.075/M, 출력 $0.30/M
- Cerebras Qwen3-235B: 무료 (프리 티어)

비용 최적화 기법:

1. 모델 계층화(Model Tiering)
- 단순 태스크 → 저비용 모델 (gpt-5-nano, Gemini Flash)
- 복잡한 판단 → 고성능 모델 (gpt-4o, Claude 3.5)
- 라우터가 태스크 복잡도에 따라 모델 선택

2. 프롬프트 캐싱(Prompt Caching)
반복 사용되는 긴 시스템 프롬프트를 캐싱:
- OpenAI: prefix 캐싱 (1,024 토큰 이상 시 자동)
- Anthropic: cache_control으로 명시적 캐싱
- 캐시 히트 시 입력 비용 50~90% 절감

3. 출력 길이 제어
- max_tokens를 태스크에 맞게 최소화
- 구체적인 길이 제한을 프롬프트에 명시
- JSON 출력은 자연어보다 토큰 효율적

4. 배치 처리(Batch Processing)
실시간이 불필요한 작업은 배치로 묶어 처리:
- OpenAI Batch API: 비실시간 작업 50% 할인
- 대량 임베딩 생성에 특히 효과적

5. 스마트 라우팅
모든 요청을 MOA로 처리하지 않음:
- 간단한 팩트 질문 → Single path (3~5배 저렴)
- 복잡한 분석 → MOA path
- 라우터 비용 < 잘못된 경로로 인한 낭비

6. 청크 크기 최적화
RAG의 청크가 너무 크면 불필요한 토큰 소비:
- 관련성 높은 부분만 선별적으로 포함
- ContextBuilder의 max_context_tokens 설정

비용 모니터링:
- 에이전트별, 경로별 비용 추적
- 일간/주간 비용 알림 설정
- 고비용 요청 패턴 분석

비용 vs 품질 트레이드오프:
단순히 비용을 줄이면 품질이 저하될 수 있습니다. A/B 테스트로 비용 절감이 품질에 미치는 영향을 정량적으로 측정한 후 결정하는 것이 중요합니다.
