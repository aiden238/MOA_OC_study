하네스 엔지니어링 개요 (AI Agent Harness)

하네스 엔지니어링(Harness Engineering)은 LLM과 외부 도구, 데이터, 에이전트들을 연결하고 조율하는 인프라와 프레임워크를 설계하는 기술 분야입니다. "하네스(Harness)"는 원래 마구(馬具)를 의미하는 단어로, LLM의 능력을 제어하고 특정 목적에 활용하기 위한 시스템 전체를 가리킵니다.

하네스의 핵심 구성 요소:
1. 오케스트레이터(Orchestrator): 전체 에이전트 실행 흐름을 제어
2. 에이전트 풀(Agent Pool): 각 역할을 담당하는 에이전트들
3. 도구 레지스트리(Tool Registry): 사용 가능한 도구 목록과 인터페이스
4. 메모리 시스템(Memory System): 컨텍스트 저장 및 검색
5. 트레이스 시스템(Trace System): 실행 기록 및 디버깅
6. 에러 핸들러(Error Handler): 실패 처리 및 복구

하네스 설계의 핵심 원칙:
1. 모듈성(Modularity): 각 컴포넌트를 독립적으로 교체 가능
2. 관찰 가능성(Observability): 모든 LLM 호출을 추적하고 기록
3. 견고성(Robustness): 개별 에이전트 실패 시 전체 파이프라인 유지
4. 비용 효율성(Cost Efficiency): 불필요한 LLM 호출 최소화
5. 지연 최소화(Low Latency): 병렬 실행과 캐싱으로 속도 개선

하네스 계층 구조:
```
[사용자 인터페이스]
       ↓
[API 게이트웨이 / 세션 관리]
       ↓
[라우터] ← [설정/정책]
       ↓
[오케스트레이터]
       ↓
[에이전트 계층: Draft / Critic / Judge / ...]
       ↓
[도구 계층: RAG / MCP / 외부 API]
       ↓
[기반 인프라: LLM API / 벡터 DB / 파일 시스템]
```

하네스 vs 프레임워크:
- LangChain/CrewAI 같은 프레임워크는 추상화 레이어 제공
- 하네스 엔지니어링은 커스텀하게 직접 구현하는 방식
- 직접 구현 시 더 세밀한 제어 가능, 의존성 최소화

Claude Code 하네스:
Claude Code는 AI 에이전트 하네스의 대표적 예시로, 코드 편집 도구, bash 실행, 파일 시스템 접근 등을 통합하여 LLM이 실제 개발 작업을 수행할 수 있도록 합니다.
