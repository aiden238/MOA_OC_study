컨텍스트 압축 및 요약 기법

컨텍스트 압축(Context Compression)은 컨텍스트 윈도우를 효율적으로 활용하기 위해 정보를 손실 최소화하면서 크기를 줄이는 기술입니다.

압축 기법 분류:

1. 추출적 압축(Extractive Compression)
- 원본 텍스트에서 중요 문장을 선택적으로 추출
- 정보 변형 없음, 원본 어휘 유지
- 예: BM25, TF-IDF 기반 중요 문장 추출
- 장점: 정확성, 단점: 맥락 단절 가능

2. 추상적 압축(Abstractive Compression)
- LLM을 사용해 내용을 이해하고 재작성
- "다음 텍스트를 핵심 정보만 유지하며 50% 압축하세요"
- 장점: 자연스러운 흐름, 단점: 정보 왜곡 가능

3. 구조적 압축(Structural Compression)
- 대화 이력을 구조화된 형태로 변환
- 예: 긴 대화 → {"topic": "X", "decisions": [...], "action_items": [...]}
- 메타데이터와 핵심 내용만 유지

대화 이력 압축 전략:
```
압축 전: 20개의 대화 메시지 (4,000 토큰)
압축 후:
  - 요약: "사용자가 X 프로젝트의 Y 기능을 개발 중, Z 오류 발생, A 방법으로 해결 시도" (200 토큰)
  - 최근 3개 메시지는 원본 유지 (600 토큰)
  총: 800 토큰 (80% 절약)
```

Progressive Summarization:
다단계 요약으로 중요도 계층을 만드는 기법:
1단계: 전체 내용 요약
2단계: 1단계에서 핵심 포인트 추출
3단계: 2단계에서 핵심 키워드 추출
→ 필요에 따라 적절한 단계의 요약 사용

RAG와 압축의 결합:
1. 청크 단위 압축: 각 청크를 미리 요약해 저장
2. 쿼리 기반 압축: 검색 쿼리와 관련된 부분만 추출
3. 계층적 RAG: 요약 → 관련 청크 → 원본 순으로 세밀도 조절

압축 품질 평가:
- 정보 보존율: 원본의 핵심 정보 유지 비율
- 압축비: 원본 대비 압축 후 크기 비율
- 다운스트림 성능: 압축 후 태스크 수행 능력 변화
