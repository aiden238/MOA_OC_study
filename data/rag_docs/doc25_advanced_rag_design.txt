고급 RAG 파이프라인 설계

기본 RAG(Retrieval-Augmented Generation)를 넘어 실용적인 고급 RAG 기법들이 개발되고 있습니다. 검색 품질과 생성 품질을 모두 향상시키는 다양한 전략이 존재합니다.

RAG 파이프라인 단계:
1. 인덱싱(Indexing): 문서 → 청크 → 임베딩 → 벡터 DB 저장
2. 검색(Retrieval): 쿼리 임베딩 → 유사도 검색 → 관련 청크 반환
3. 증강(Augmentation): 검색된 청크를 프롬프트에 포함
4. 생성(Generation): 증강된 프롬프트로 LLM 답변 생성

고급 검색 기법:

1. 하이브리드 검색(Hybrid Search)
- 키워드 검색(BM25) + 의미 검색(벡터) 결합
- RRF(Reciprocal Rank Fusion)으로 결과 통합
- 정확 매칭과 의미적 유사성을 동시에 활용

2. HyDE (Hypothetical Document Embeddings)
- 쿼리로 가상의 답변을 먼저 생성
- 가상 답변의 임베딩으로 실제 문서 검색
- 쿼리와 문서 임베딩 공간의 차이를 보완

3. Multi-Query Retrieval
- 하나의 질문을 여러 다른 표현으로 변환
- 각 표현으로 독립 검색 후 결과 합산
- 쿼리의 다양한 측면을 포괄

4. Contextual Compression
- 검색된 청크에서 쿼리와 관련된 부분만 추출
- 관련성 낮은 내용을 제거해 컨텍스트 효율화
- LLM Compressor 또는 규칙 기반 필터 사용

5. Parent-Child Chunking
- 검색은 작은 청크(child)로, 컨텍스트는 큰 청크(parent)로
- 정밀한 검색 + 충분한 맥락 제공의 균형

재순위(Reranking):
초기 검색 결과의 순서를 더 정확한 모델로 재조정:
- Cross-encoder 모델: 쿼리-문서 쌍을 함께 평가
- Cohere Rerank, BGE Reranker 등 활용
- 상위 k개를 더 많이 검색 후 재순위로 상위 n개 선택

RAG 평가 지표:
- Context Relevance: 검색된 문서가 쿼리와 관련 있는가
- Faithfulness: 생성된 답변이 검색된 문서에 근거하는가
- Answer Relevance: 최종 답변이 원래 질문에 답하는가

RAGAS 프레임워크로 이 세 지표를 자동 평가할 수 있습니다.
