지침 파일 구조 설계 (CLAUDE.md, 설정 파일)

지침 파일(Instruction File)은 AI 에이전트 시스템에서 에이전트의 행동 방식, 프로젝트 컨텍스트, 제약 조건 등을 정의하는 문서입니다. CLAUDE.md, AGENTS.md, .cursorrules 등 다양한 이름으로 활용됩니다.

지침 파일의 목적:
1. 에이전트에게 프로젝트 고유의 컨텍스트 전달
2. 일관된 행동 방식 보장
3. 자주 반복되는 지시를 파일로 추출해 재사용
4. 팀 간 AI 사용 가이드라인 공유

효과적인 지침 파일 구조:
```
# [프로젝트명] — AI 에이전트 지침

## 프로젝트 정체성
[한 줄 정의와 핵심 목표]

## 아키텍처 개요
[시스템 구조 다이어그램 또는 설명]

## 가드레일
[절대 지켜야 할 제약 조건 목록]

## 커밋/코딩 컨벤션
[프로젝트별 규칙]

## 디렉토리 구조
[주요 폴더와 역할 설명]

## 참조 파일
[더 자세한 정보가 담긴 파일 목록]
```

계층적 지침 파일 패턴:
- 루트 CLAUDE.md: 전역 프로젝트 지침
- 모듈별 CLAUDE.md: 특정 디렉토리에 적용되는 세부 지침
- refs/ 디렉토리: 기술 스택, 평가 프레임워크 등 세부 문서

지침 파일 설계 원칙:
1. 계층 분리: 자주 바뀌는 내용과 안정적인 내용 분리
2. 참조 분리: 긴 내용은 별도 파일로 분리하고 링크
3. 우선순위 명시: 가드레일 같은 절대 원칙을 표에 정리
4. 상태 추적: 진행 중인 작업 상태를 파일에 포함
5. 자기 설명적: 새 에이전트가 읽어도 이해할 수 있도록

동적 지침 vs 정적 지침:
- 정적 지침: 거의 변하지 않는 프로젝트 규칙 (가드레일, 컨벤션)
- 동적 지침: 작업 진행에 따라 업데이트되는 상태 (진행률, 현재 작업)

지침 파일과 시스템 프롬프트의 관계:
지침 파일은 에이전트 세션 시작 시 시스템 프롬프트에 포함되거나 RAG 방식으로 검색되어 적재됩니다. Claude Code의 경우 CLAUDE.md를 자동으로 컨텍스트에 포함시킵니다.
