역할 기반 프롬프팅 (Role Prompting)

역할 기반 프롬프팅(Role Prompting)은 LLM에게 특정 전문가나 페르소나의 역할을 부여하여 해당 역할에 맞는 스타일, 어조, 전문성을 갖춘 응답을 유도하는 기법입니다.

기본 구조:
"당신은 [역할/전문가]입니다. [배경 설명]. [태스크]를 수행하세요."

역할 부여의 효과:
1. 응답 일관성 향상: 동일한 전문성 수준을 유지
2. 어조/스타일 조절: 기술적, 대중적, 공식적 등 상황에 맞는 표현
3. 도메인 지식 활성화: 해당 분야의 지식을 더 잘 활용
4. 창의성 조절: 특정 제약 내에서 창의적 작업 수행

효과적인 역할 설계:
- 구체적일수록 좋음: "전문가" → "10년 경력의 백엔드 시니어 엔지니어"
- 제약 조건 포함: "초등학생도 이해할 수 있는 설명 방식으로"
- 목표 명시: 역할이 달성해야 할 최종 목적 포함
- 행동 지침: 해야 할 것과 하지 말아야 할 것을 명시

시스템 프롬프트에서의 역할 부여:
```
시스템: "당신은 MOA(Mixture of Agents) 파이프라인의 분석적 초안 작성 에이전트입니다.
         논리적이고 구조적인 방식으로 정보를 분석하여 초안을 작성합니다.
         감정적 표현을 지양하고 데이터와 근거 중심으로 서술합니다."
```

멀티 에이전트 시스템에서의 활용:
역할 프롬프팅은 각 에이전트에 서로 다른 관점과 전문성을 부여하는 데 핵심적입니다.
- Draft Agent: 특정 관점(분석적/창의적/구조적)의 역할 부여
- Critic Agent: 객관적 평가자 역할
- Synthesizer Agent: 통합 및 합성 전문가 역할
- Judge Agent: 품질 심사관 역할
