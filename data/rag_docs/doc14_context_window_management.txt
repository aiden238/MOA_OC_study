컨텍스트 윈도우 관리

컨텍스트 윈도우(Context Window)는 LLM이 한 번에 처리할 수 있는 최대 토큰 수입니다. GPT-4o는 128K, Claude 3.5는 200K, Gemini 1.5 Pro는 1M 토큰까지 지원하지만, 실용적으로 전체를 사용하는 것은 비용과 성능 면에서 비효율적입니다.

토큰 예산 분배 전략:
전체 토큰 예산(예: 8,000 토큰) 분배 예시:
- 시스템 프롬프트: ~500 토큰 (6%)
- RAG 컨텍스트: ~2,000 토큰 (25%)
- 대화 이력: ~2,000 토큰 (25%)
- 현재 질문: ~200 토큰 (2.5%)
- 예약(출력): ~3,300 토큰 (41%)

컨텍스트 오버플로우 처리 전략:
1. 슬라이딩 윈도우: 오래된 메시지를 순서대로 제거
2. 요약 압축: 긴 대화 이력을 LLM으로 요약
3. 중요도 기반 삭제: 관련성 낮은 내용 우선 제거
4. 청크 분할: 긴 문서를 작은 청크로 분할해 필요한 것만 포함

토큰 최적화 기법:
1. 프롬프트 압축: 불필요한 공백, 반복 표현 제거
2. 약어 활용: 반복되는 긴 표현을 짧게 처리
3. JSON 압축: 들여쓰기 없는 최소화된 JSON 사용
4. 선택적 포함: 현재 태스크와 무관한 이력 배제
5. 캐싱 활용: OpenAI Prompt Cache, Anthropic Cache 활용

컨텍스트 품질 지표:
- 관련성 비율: 전체 토큰 중 현재 태스크에 직접 기여하는 비율
- 정보 밀도: 단위 토큰당 유용한 정보량
- 중복률: 동일하거나 유사한 내용의 반복 비율

멀티턴 대화에서의 관리:
대화가 길어질수록 초기 컨텍스트가 희석됩니다. 이를 방지하기 위해:
- 핵심 지침을 주기적으로 재주입
- 대화 중반에 중간 요약 생성
- 사용자 의도와 진행 상태를 별도 저장
